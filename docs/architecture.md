# Architecture & Design

이 문서는 터미널 체스 프로그램의 데이터 구조체 설계와 핵심 함수들의 역할 분담을 기록합니다.
## 0. 원하는 구현 방식
- 1. 프로그램을 실행하면, 흑과 백의 닉네임을 입력을 하게 된다.
- 2. 입력을 하면, 체스 게임이 시작이 되고, 백부터 교대로 설계하고, 체크메이트 감지시, 게임은 종료된다.

## 2. 데이터 구조 설계 (Data Structures)


### 2.1 말(Piece) 및 보드 데이터
체스판의 각 칸과 말의 상태를 표현하기 위해 열거형(Enum)과 9×9배열로 설계했습니다.(index=0인 부분은 안쓰는 부분)

```c
typedef enum { 
    NONE, 
    PAWN, 
    KNIGHT, 
    BISHOP, 
    ROOK, 
    QUEEN, 
    KING } piecetype;
char board[BOARDSIZE+1][BOARDSIZE+1];
```
### 2.2 그 외 필요한 변수들
| 변수명 | 의미 |예시|
| --- | --- | --- |
| turn | 흑백 차례를 알려줌|0이면 백, 1이면 흑|
| isCheckmate | 현재 상태가 누구의 체크메이트 상태인지를 알려줌. |0이면 정상, 1이면 백의 체크, 2이면 흑의 체크, 3이면 스테일메이트

### 3.2 게임 시작
닉네임을 입력받고, 게임을 백부터 순서대로 진행한다.
* **서명:** `void startHandler(nickname[2][20]);`
* **매개변수:** 
nickname[2][20]: 각 유저의 닉네임
* **반환값:** 없음 (`void`)

있으면 1, 없으면 0
